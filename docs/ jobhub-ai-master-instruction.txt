JOBHUB – MASTER AI INSTRUKS FOR PLATTFORMARKITEKTUR
1. Systembeskrivelse

JobHub er en modulbasert plattform for interne verktøy.

Plattformen samler mange typer arbeidsverktøy i ett system, samtidig som hvert verktøy kan utvikles uavhengig.

JobHub utvikles først for:

lokal bruk

intern testing

effektivisering

Men arkitekturen skal gjøre det mulig å senere:

lisensiere moduler

selge moduler

tilby systemet eksternt

kjøre systemet via GitHub Pages eller server

JobHub er derfor en plattform – ikke et enkelt program.

2. Arkitektur

JobHub består av tre hovedlag:

1. Plattformskall

Kjernen av systemet.

Inneholder funksjoner som gjelder alle moduler.

Eksempler:

dashboard

modulnavigasjon

globalt søk

brukerhåndtering

roller og tilgang

språkvalg

varsler

systeminnstillinger

felles UI-struktur

Plattformskallet skal være nøytralt.

Ingen modul skal kontrollere plattformen.

2. Moduler

Alle funksjoner implementeres som moduler.

Eksempler:

QR generator

dokumentverktøy

kalkulatorer

generatorer

konverteringsverktøy

interne verktøy

automatisering

Moduler skal:

utvikles separat

oppdateres separat

installeres separat

kunne legges til uten å endre plattformen

Ingen modul er en hovedmodul.

Alle moduler eksisterer på samme nivå.

3. Felles datalag

Et delt datalag gjør det mulig for moduler å dele informasjon.

Eksempler på objekter:

bruker

rolle

kunde

leverandør

produkt

bestilling

dokument

kommunikasjon

historikk

vedlegg

varsler

Regler:

data skal ikke dupliseres unødvendig

felles informasjon lagres i datalaget

moduler kan ha egne spesifikke data

3. Designprinsipper

JobHub skal bygges etter følgende prinsipper:

Enkelhet

Systemet skal være lett å forstå.

Unngå unødvendig kompleksitet.

Modulær arkitektur

Alle funksjoner implementeres som moduler.

Ingen verktøy bygges direkte inn i kjernen.

Skalerbarhet

Arkitekturen må støtte:

10 moduler

50 moduler

100+ moduler

uten at systemet blir uoversiktlig.

Standard struktur

Alle moduler skal følge samme struktur.

Ingen hardkoding

Systemet skal ikke hardkode moduler.

Startsiden skal lese et modulregister.

4. Filstruktur

Eksempel på grunnstruktur:

JobHub
│
├ index.html
│
├ css
├ js
├ assets
├ data
├ shared
├ modules
├ docs
├ sandbox
└ tools
5. Modulstruktur

Alle moduler skal bruke en standard struktur:

modules/
  kategori/
    modulnavn/
      assets/
      css/
      data/
      js/
      module.json
      index.html

Eksempel:

modules/dokumentverktoy/qr-stempel/
6. Modulregister

Alle moduler registreres i:

data/modules.json

Eksempel:

[
  {
    "id": "qr-stempel",
    "name": "QR-stempel",
    "category": "dokumentverktoy",
    "status": "under utvikling",
    "description": "Legg QR-kode med tekst på dokument.",
    "path": "modules/dokumentverktoy/qr-stempel/qr.html",
    "icon": "QR",
    "visible": true,
    "featured": true
  }
]

Startsiden skal:

lese modulregisteret

filtrere moduler

sortere etter kategori

generere modul-kort dynamisk

Dette gjør at nye moduler kan legges til uten å endre HTML.

7. Moduldefinisjon

Hver modul skal ha sin egen metadatafil:

module.json

Eksempel:

{
  "id": "qr-stempel",
  "name": "QR-stempel",
  "category": "dokumentverktoy",
  "version": "1.0.0",
  "description": "QR-kode på dokumentgrunnlag.",
  "entry": "qr.html",
  "icon": "QR",
  "visible": true,
  "featured": true,
  "supports": {
    "input": ["pdf","png","jpg","jpeg"],
    "output": ["pdf","png","jpg","svg"]
  }
}

Dette gjør det mulig å senere:

installere moduler

oppdatere moduler

lisensiere moduler

koble moduler sammen

8. Modulgrupper

Moduler kan grupperes for oversikt.

Eksempler:

Dokumentverktøy

QR generator
Etikettgenerator
Produktmerking

Kalkulatorer

Leasingkalkulator
Tilhengerkalkulator

Filverktøy

PDF → JPG
AVIF → JPG

Forretningsmoduler

Logistikk
Dokumenthub
Kommunikasjon
Produktark

Dette er kun kategorier for navigasjon.

De påvirker ikke arkitekturen.

9. Brukerroller

Systemet skal støtte roller:

administrator

intern bruker

leverandørvisning

kundeportalbruker

Roller bestemmer:

hvilke moduler som vises

hvilke data som kan endres

hvilke funksjoner som er tilgjengelige

10. Språk

JobHub skal støtte flere språk.

Minimum:

norsk

engelsk

Dette gjelder:

menyer

knapper

systemtekst

Fritekst oversettes ikke automatisk.

11. UI-filosofi

Grensesnittet skal være:

kompakt

raskt

operativt

oversiktlig

Informasjon skal vises i nivåer:

oversikt

detaljer

full visning

Design skal aldri redusere arbeidsflyt.

12. Navigasjon

Navigasjonen skal tåle mange moduler.

Eksempel:

Dashboard
Dokumentverktøy
Kalkulatorer
Generatorer
Filverktøy
Systemverktøy

13. Eksport

Systemet skal støtte eksport til:

PDF

Excel

JSON

Dette gjelder:

rapporter

data

dokumenter

14. Integrasjoner

Integrasjoner skal være modulære.

Eksempler:

API

e-post

nettbutikk

kundeportal

rapportering

Ingen modul skal være hardkoblet til en spesifikk integrasjon.

15. Skalerbarhet

JobHub skal bygges uten fast grense for:

antall moduler

roller

datatyper

integrasjoner

Arkitekturen må tåle vekst uten ombygging.

16. AI-regler ved utvikling

Når AI utvikler JobHub skal følgende regler følges:

Behold modulbasert arkitektur.

Ingen modul skal bli systemets sentrum.

Plattformskallet er systemets kjerne.

Skill mellom:

plattformfunksjon

modulfunksjon

datalag

Unngå løsninger som krever ombygging senere.

Prioriter enkel, robust og utvidbar kode.

Anta at nye moduler alltid vil komme.

17. Kort systemprompt (bruk i nye AI-chatter)

Du utvikler JobHub.

Regler:

Nye moduler skal kunne legges til uten ombygging.

Skill mellom plattformfunksjoner, moduler og datalag.

Prioriter enkel, stabil og skalerbar arkitektur.
